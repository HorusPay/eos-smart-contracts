Definitions and notes on the Bancor Algorithm:

  • TOKTOK is the your native Smart Token symbol, which could be HORUS for example
  • CTOKCTOK is the connector token that you wish to transfer, which could be EOS, BTC, or even ETH, if desired.
  • [USD/TOK] the brakets indicate only the units of the variable, in this case the unit is USD/TOK
  • From the whitepaper, "The price of a Smart Token is denominated in the connected token (for example, the price of BNT is denominated in ETH, its connected token"
  • 'The bancor formula is built upon the idea that the smart contract maintains a ratio between the total value and the connector valance, the connector weight.'

Total Smart Token Supply

The Smart Token Total Supply, totalTOKsupplytotalTOKsupply, is the maximum amount of Smart Tokens of your contract to ever be created.

Smart Token Market Cap

The total market capitalization of the Smart Token, TOKTOK, is the total value of the TOKTOK market, in reference of the connector token, CTOKCTOK. The TOKmarketCapTOKmarketCap variable must be in units of [CTOK] for CWCW to be a unitless number. Therefore, the whitepaper formula for TOKmarketCapTOKmarketCap should be modified to be:

TOKmarketcap[CTOK]=price[CTOK/TOK]totalTOKsupply[TOK]TOKmarketcap[CTOK]=price[CTOK/TOK]totalTOKsupply[TOK]

For instance, if the total supply of EOS ever created was 1000 EOS, than the TOKmarketcapTOKmarketcap would be:

TOKmarketcap=0.00074823558[BTC/EOS]1000EOS=0.74680BTCTOKmarketcap=0.00074823558[BTC/EOS]1000EOS=0.74680BTC
.

Connector Weight

From the Price Formula algorithms:

CW=CTOKbalance[CTOK]TOKmarketcap[CTOK]CW=CTOKbalance[CTOK]TOKmarketcap[CTOK]

where:

Where, CTOKbalanceCTOKbalance is the balance of the tokens in the connector account.

A few small notes:

  • The whitepaper states that: "The CW is expressed as a percentage greater than 0% and up to 100%, and as mentioned, is initially set by the creator when a Smart Token is configured."

If this were true, and CTOKbalanceCTOKbalance and TOKmarketcapTOKmarketcap are of the same units, than the equation for CWCW would have to be:

CW=CTOKbalance[CTOK]TOKmarketcap[CTOK]100CW=CTOKbalance[CTOK]TOKmarketcap[CTOK]100

But this would scale priceprice calculation to be artificially low (I think), so it is probably ranging between 0<CW<10<CW<1.

Smart Token Price

The price of the smart token is denominated (referrenced to) the connector token. For example, if EOS is the Smart token and BTC is the connector token, and 1 EOS = 0.00074823558 BTC, than the units of priceprice are:

price=0.00074823558[BTC/EOS]price=0.00074823558[BTC/EOS]
.

In reference to this algorithm, the units of price are.

price=XX.XXXX[CTOK/TOK]price=XX.XXXX[CTOK/TOK]

Solve for the Smart Token's price as a function of it's connector weight, CWCW, connector balance, CTOKbalanceCTOKbalance, and Smart Token supply, TOKsupplyTOKsupply. The SmartTokenOutstandingSupplySmartTokenOutstandingSupply is not defined in the Bancor whitepaper, but we're assuming it represents available supply of TOKTOK to the smart contract, TOKsupplyTOKsupply. This TOKsupplyTOKsupply would have to be a liquid balance of TOKTOK, owned by the Smart Contract administrators, to be usefully traded.

price[CTOK/TOK]=CTOKBalance[CTOK]TOKsupply[TOK]CWprice[CTOK/TOK]=CTOKBalance[CTOK]TOKsupply[TOK]CW

This formula implies that if there is a large outstanding supply (low demand), the token will be worth a fraction of it's marketcap price and if only a few tokens are available in supply (high demand), each token will be worth nearly the marketcap price.

Tokens Issued

TOKissued[TOK]=TOKsupply[TOK]((1+(CTOKrecieved[CTOK]CTOKbalance[CTOK])CW1)TOKissued[TOK]=TOKsupply[TOK]((1+(CTOKrecieved[CTOK]CTOKbalance[CTOK])CW1)

This is the amount of Smart Tokens to payout given the current smart token supply, TOKsupplyTOKsupply, the connected tokens paid, CTOKrecievedCTOKrecieved, and the balance of CTOKCTOK tokens in the bancor account, CTOKbalanceCTOKbalance.

CTOKissued=CTOKbalance((1+TOKrecievedTOKsupply)1CW1)CTOKissued=CTOKbalance((1+TOKrecievedTOKsupply)1CW1)

The effective price can be calculated as:

effectivePrice[CTOK/TOK]=CTOKexchanged[CTOK]TOKexchanged[TOK]effectivePrice[CTOK/TOK]=CTOKexchanged[CTOK]TOKexchanged[TOK]

The effectivePriceeffectivePrice represents the effective exchange rate of the transaction, in units of [CTOK/TOK][CTOK/TOK]

class smart_token:
    def __init__(self, name, supply, totalsupply):
        self.name = name                     #"TOK"
        self.supply = supply                 #[TOK] circulating supply
        self.totalsupply = totalsupply       #[TOK] total TOK created
    
    def printtok(self):
        print("Token Stats:")
        print("Token: ", self.name)
        print("Supply: ", self.supply)
        print("totalsupply: ",  self.totalsupply)
        
    
class connector:
    def __init__(self, tokenname, balance):
        self.token   = tokenname   # name="CTOK", for example
        self.balance = balance     # [CTOK] balance

        
    def printcon(self):
        print("Connector Stats:")
        print("Token: ", self.token)
        print("Balance: ", self.balance)

    
class BancorExchange:
    
    def __init__(self, CW):
        self.CW = CW
        self.price = 0.0
        self.effective_price = 0.0

    def printf(self):
        print("CW: ", self.CW)
        print("Price: ", self.price)
        
    def CTOKtoTOK(self, tok, ctok, CTOKreceived):
        
        # get TOK supply
        TOKsupply = tok.totalsupply
        
        # get price
        self.price = self.getprice(tok, ctok)
        
        # get TOKmarketcap
        marketcap = self.marketcap(self.price, tok.totalsupply)
        
        # compute TOKissued
        TOKissued = self.TOKissued(TOKsupply, ctok.balance, CTOKreceived, CW)
        
        # compute the effective price
        
        # remove TOK from TOKbalance
        tok.supply -= TOKissued
        
        # add CTOK to CTOKbalance
        ctok.balance += CTOKreceived
        
        # calculate the effective price
        self.effective_price =   CTOKreceived/ (1.0*TOKissued)
    
        # update the new marketcap
        newmarketcap = self.marketcap(self.effective_price, tok.totalsupply)
        
        # update the CW
        self.CW = self.updateCW(ctok.balance, newmarketcap)
        
        # return TOKissued and effective price
        return (TOKissued)
    
    def TOKtoCTOK(self, tok, ctok, TOKreceived):
        
        
        # get price
        self.price = self.getprice(tok, ctok)
        
        # get TOKmarketcap
        marketcap = self.marketcap(self.price, tok.totalsupply)
        
        # compute CTOKissued
        CTOKissued = self.CTOKissued(tok.totalsupply, ctok.balance, TOKreceived, CW)
        
        # remove TOK from TOKbalance
        ctok.balance -= CTOKissued
        
        # add CTOK to CTOKbalance
        tok.supply += TOKreceived
        
        # calculate the effective price
        self.effective_price =   CTOKissued/ (1.0*TOKreceived)
    
        # update the new marketcap
        newmarketcap = self.marketcap(self.effective_price, tok.totalsupply)
        
        # update the CW
        self.CW = self.updateCW(ctok.balance, newmarketcap)
        
        # return TOKissued and effective price
        return (CTOKissued)
        
        
    
        
    def updateCW(self, CTOKbalance, TOKmarketcap):
        CW = CTOKbalance / TOKmarketcap
        return (CW)
        
    def getprice(self, tok, ctok):
        balance   = ctok.balance
        TOKsupply = tok.supply
        price = balance/(TOKsupply*self.CW)
#         print("Price [EOS/HOR]:", price)
        return ( price )
    
    def marketcap(self,price, totalTOKsupply):
        marketcap = price * totalTOKsupply
        return ( marketcap )
    
    def CTOKissued(self, TOKsupply, CTOKbalance, TOKreceived, CW ):
        
        ctokissued = CTOKbalance * (( 1 + TOKreceived/TOKsupply)**(1/CW)-1 )
 
        return ( ctokissued )

    def TOKissued(self, TOKsupply, CTOKbalance, CTOKreceived, CW):
        
        tokensissued = TOKsupply*( (1+CTOKreceived/CTOKbalance)**(CW) -1)

        return( tokensissued )
